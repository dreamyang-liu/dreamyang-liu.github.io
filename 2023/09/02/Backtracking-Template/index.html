Backtracking

Backtracking is normally used to search the solution space. Such solution space normally is combinatorial solution space which increase exponentially. A good example is the permutation problem.

Given a number \(n\), print out all the permutations of numbers of \([1,\dots, n]\).

For this problem, \(n = 3\), the expected output should be

In this case, we need to use backtrack to help us tackling this problem. Backtrack can save the checkpoint every time when we need to make a decision on which candidate to expand.

Here is a template for Backtracking.

Let's do a partial dry run for the permutation problem.

  1. We pick \(1\) from candidates and remove \(1\) from it
  2. We pick \(2\) from candidates and remove \(2\) from it
  3. We pick \(3\) from candidates and remove \(3\) from it
  4. candidates is empty and we jump out from backtrack # [1,2,3]
  5. We add \(3\) back to candidates and find there's no other candidate.
  6. We add \(2\) back to candidates and move to the next candidate 3
  7. We pick \(3\) from candidates and remove \(3\) from it
  8. We pick \(2\) from candidates and remove \(2\) from it
  9. candidates is empty and we jump out from backtrack # [1,3,2]
  10. \(\dots\)
